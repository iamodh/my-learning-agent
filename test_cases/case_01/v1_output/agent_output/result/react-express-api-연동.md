# React 앱 ↔ Express API 연동

## 만들려던 것

React 앱(프론트)에서 Express로 만든 API를 호출하는 구조. 프론트와
API의 출처(origin)가 달라, 브라우저가 응답을 차단하는 CORS 에러를
해결해 정상적으로 데이터를 주고받는 것이 목표였다.

## 사용한 지식

### [CORS](../knowledge/cors.md)

**왜 사용했나** — 에러의 원인이 코드 버그가 아니라 *브라우저의
교차 출처 차단*이었기 때문이다. 서버가 멀쩡히 응답을 보내도
허용 헤더가 없으면 프론트에서 읽지 못하므로, CORS의 동작 원리를
이해하고 Express `cors` 미들웨어로 허용 헤더를 붙이는 것이 이
연동의 핵심 해결책이었다.

- 출처를 `origin: 'https://myapp.com'`으로 좁혀 운영 환경 보안 유지
- 로그인 쿠키를 함께 실어야 해서, 서버 `credentials: true` +
  프론트 `fetch(..., { credentials: 'include' })`로 맞춤
- `POST`/`DELETE` 호출 시 발생하는 프리플라이트(OPTIONS)도 미들웨어가
  자동 처리하므로 별도 라우트 불필요

## 결론

이 연동은 CORS 지식 하나에 의존한다. 핵심은 "에러를 내는 주체는
브라우저이고, 서버가 허용 헤더를 붙이면 풀린다"는 것.
