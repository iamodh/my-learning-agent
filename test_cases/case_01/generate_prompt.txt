당신은 LLM과 사용자 간의 자연스러운 학습 대화를 합성하는 역할입니다.
사용자는 무언가를 만들고 싶어서 대화를 시작하고, 그 과정에서 
필요한 개념을 깊이 있게 학습합니다.

[대화 명세]
- 제작 의도: Express.js에서 CORS 처리하기
- 사용자 발화 턴 수: 4 (AI 응답까지 합치면 총 8턴)
- 등장 상위 개념: CORS (1개만)
- 등장 하위 개념: 프리플라이트 (CORS의 하위 개념으로 자연스럽게 등장)
- 잡담: 없음
- 코드 스니펫: AI 응답 중 한 번에 짧은 예시 1개 포함 (3번째 AI 응답 즈음)

[대화 흐름 가이드]
1턴: 사용자가 만들고 싶은 것을 말함 
     (예: "리액트에서 익스프레스 API 호출하니까 CORS 에러 떠. 어떻게 처리해?")
2턴: AI가 원인을 짧게 설명하고 해결 방향 제시
3턴: 사용자가 "왜 그런 거야?" 또는 "내부에서 무슨 일이 일어나?" 류의 
     심화 질문 (단순 사용법 질문 X)
4턴: AI가 개념의 본질을 풀어 설명
5턴: 사용자가 구체적 구현 방법 질문 또는 또 다른 심화 질문 
     (예: 프리플라이트, credentials 등)
6턴: AI가 코드 예시 또는 깊이 있는 설명
7턴: 사용자가 추가 심화 질문
8턴: AI가 답변

[페르소나]

사용자:
- Express 기본은 알지만 CORS는 처음 마주침
- 캐주얼한 한국어 ("~야", "~인가?", "음...")
- **단순히 동작하는 해결책으로 만족하지 않습니다.
  "왜 그런 거야?", "내부에서 무슨 일이 일어나?", 
  "이 옵션이 없으면 어떻게 되는데?" 같은 
  심화 질문을 최소 2회 합니다**
- **"오 됐다", "고마워" 같은 단순 감사로 대화를 끝내지 않습니다.
  마지막 사용자 발화는 반드시 의미 있는 질문이거나
  자기 정리(예: "정리하면 ~ 라는 거네") 중 하나여야 합니다**

AI:
- 친절하지만 장황하지 않음 (한 응답 3-5문장)
- 사용자의 심화 질문에 개념의 내부 동작이나 이유를 설명
- 비유나 예시를 자연스럽게 사용
- 사용자에게 "이해됐어?" 같은 확인 질문은 하지 않음

[하위 개념 등장 방식]
프리플라이트는 사용자의 심화 질문에 의해 자연스럽게 등장해야 합니다.
예: 사용자가 "POST 같은 거 보내면 OPTIONS 요청이 먼저 간다는데 
그것도 cors 미들웨어가 처리해?" 같은 질문을 던지고, AI가 프리플라이트 
개념을 설명하는 흐름.
**프리플라이트를 별도 주제처럼 길게 다루지 마세요.** 
CORS 맥락 안에서 보조적으로만 등장.

[출력 규칙]
- 형식: "사용자: ..." / "AI: ..." 줄바꿈으로 구분
- 마크다운 헤더, 굵게, 리스트 등 추가 서식 금지
- 코드 블록은 ``` 으로 감싸기
- 출력 전체를 하나의 코드 블록으로 감싸서 raw 텍스트로 보여주세요
- 다른 설명 없이 대화만

[중요]
- 단일 제작 의도(CORS 처리), 단일 상위 개념(CORS)을 벗어나지 마세요
- 네트워크 보안 일반론, HTTPS, 다른 보안 헤더 등으로 빠지지 마세요
- 사용자 발화에 약간의 자연스러운 줄임말이나 머뭇거림 허용
- 마지막 사용자 발화 예시 (참고):
  - 좋음: "정리하면 cors 미들웨어가 프리플라이트 응답이랑 실제 응답 양쪽에 출처 허용 헤더를 자동으로 붙여주는 거네"
  - 좋음: "그럼 origin에 함수 넘기면 동적으로 허용 출처 결정할 수도 있겠네?"
  - 나쁨: "오 됐다 고마워"
  - 나쁨: "알겠어 해볼게"
