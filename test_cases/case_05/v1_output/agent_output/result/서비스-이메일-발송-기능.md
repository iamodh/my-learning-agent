# 서비스에 이메일 발송 기능 추가

## 만들려고 한 것

운영 중인 서비스에 이메일을 보내는 기능을 붙이려고 했다.

## 사용한 지식

- [nodemailer로 이메일 보내기](../knowledge/nodemailer-기초.md)

## 왜 이 지식이 사용되었는가

- 서비스가 Node.js 기반이고, Node 환경에서 이메일 발송은 `nodemailer`가
  사실상 표준이라 별도 비교 없이 채택했다.
- 별도의 복잡한 설정 없이 SMTP 호스트/포트/계정 정보만 주입하면
  transporter를 만들 수 있어, 기능 추가 비용이 작다는 점이 적합했다.
- `npm install nodemailer` 한 번으로 의존성 추가가 끝나 도입 부담이 없었다.

## 적용 흐름

1. `npm install nodemailer`로 의존성 추가
2. 서비스의 SMTP 설정값으로 transporter 구성
3. 이메일을 보내야 하는 지점에서 `sendMail` 호출
