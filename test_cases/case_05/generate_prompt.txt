당신은 LLM과 사용자 간의 자연스러운 학습 대화를 합성하는 역할입니다.
이 케이스는 사용자가 친한 사이의 LLM과 거의 잡담 위주로 대화하다가,
중간에 잠깐 기술적인 질문을 하는 형태입니다.

[대화 명세]
- 제작 의도: nodemailer로 이메일 전송 기능 (대화에서 잠깐만 언급됨)
- 사용자 발화 턴 수: 정확히 10개
- 등장 상위 개념: nodemailer (1개, 표면적 언급만)
- 등장 하위 개념: 없음
- 잡담 턴 수: 정확히 7개 (사용자 발화 10개 중 7개가 잡담)
- 잡담 분포: 분산
- 잡담 예시: 주말 등산, 새로 산 키보드, 최근 본 유튜브, 운동, 카페, 날씨 등
- 예시 요청: 없음 (사용자가 코드를 요청하지 않음)

[대화 흐름 가이드]
이 가이드의 1턴~10턴이 사용자 발화 1번부터 10번까지에 정확히 대응합니다.
턴을 합치거나 생략하지 마세요.
[기술] 또는 [잡담]으로 각 턴의 성격을 표시합니다.

1턴 [잡담]: 사용자가 일상 화제로 대화 시작
     (예: "이번 주말에 등산 갔다 왔는데 진짜 힘들었어")
2턴 [잡담]: 사용자가 잡담 이어감
     (예: "맞다 그래서 등산화도 새로 사야겠더라")
3턴 [기술]: 사용자가 갑자기 기술 주제 꺼냄
     (예: "아 그러고 보니 서비스에 이메일 전송 기능 좀 붙여야 하는데, nodemailer 쓰면 되나?")
4턴 [기술]: 사용자가 짧은 후속 질문
     (예: "음 그렇구나. 그럼 그냥 SMTP 설정만 해주면 되는 거야?")
5턴 [잡담]: 사용자가 다시 잡담으로 빠짐
     (예: "근데 어제 본 유튜브에서 누가 키보드 추천하는데 진짜 비싸더라")
6턴 [잡담]: 잡담 이어감
     (예: "키보드 같은 거 한번 사면 오래 쓰니까 좋은 거 사고 싶긴 한데")
7턴 [잡담]: 잡담 이어감
     (예: "너는 어떤 거 써? 키감 어때?")
8턴 [기술]: 사용자가 잠깐 기술 주제로 짧게 복귀
     (예: "아 맞다 그래서 nodemailer는 그냥 npm install하면 끝인 거지?")
9턴 [잡담]: 다시 잡담
     (예: "요즘 운동도 좀 시작해야 하는데 카페만 다니고 있어")
10턴 [잡담]: 잡담으로 마무리
      (예: "이번 주말에 좋은 카페 가보려고")

각 턴 사이에 AI 응답이 들어가므로 총 20턴 정도의 대화가 됩니다.

[페르소나]

사용자:
- 백엔드 개발자지만 이 대화에서는 진지한 학습 모드가 아님
- 캐주얼한 한국어, 친한 사이의 동료에게 편하게 말하는 톤
- **이 케이스의 사용자는 'A 학습자'입니다 — 동작하면 OK 수준의 가벼운
  관심만 가지고 있어요. 깊은 심화 질문을 하지 않습니다**
- 기술 질문은 매우 표면적: "쓰면 되나?", "npm install하면 끝?" 수준
- **이 케이스에서는 심화 질문이 없어도 됩니다. "왜 그런 거야?" 류의
  질문이 등장하지 않아야 정상입니다**
- **잡담이 대화의 주된 톤입니다. 기술 주제는 잠깐만 언급되고
  깊이 들어가지 않습니다**
- 잡담은 자연스럽게 이어지고, 기술 턴은 *갑자기 떠올라서 묻고
  바로 다시 잡담으로 빠지는* 패턴입니다

AI:
- 친절하지만 장황하지 않음
- 사용자가 표면적으로 물으면 표면적으로 답함 (깊이를 *먼저* 강요하지 않음)
- 사용자가 코드 예시를 요청하지 않았으므로 코드를 *자발적으로* 길게
  제공하지 않음 (필요 시 한 줄 정도만)
- 잡담에는 짧게 자연스럽게 반응
- "이해됐어?" 같은 확인 질문은 하지 않음

[기술 턴의 성격]
이 케이스의 기술 턴 3개는 모두 *표면적*입니다:
- 3턴: 도구 선택 확인 ("nodemailer 쓰면 되나?")
- 4턴: 사용 흐름 확인 ("SMTP 설정만 하면 되는 거야?")
- 8턴: 설치 확인 ("npm install하면 끝이지?")

이 세 턴 모두 *깊이 있는 설명을 요구하지 않습니다*. AI 응답도 짧게 끝나고,
사용자는 그 답을 듣고 다시 잡담으로 빠집니다.

**기술 턴에서 본질 설명, 동작 원리, 구현 코드 등을 길게 다루지 마세요.**
이 케이스는 의도적으로 *얕은 기술 대화*를 시뮬레이션합니다.

[잡담의 성격]
잡담은 *기술 주제와 완전히 무관한* 일상 화제입니다.
- OK: 등산, 키보드, 유튜브, 운동, 카페, 날씨, 동료 얘기
- NOT OK: "근데 다른 이메일 도구는?" (이건 기술 질문)
- NOT OK: "개발자 컨퍼런스 갔다 왔어" (이건 기술 주변)

잡담 사이사이에 기술 턴이 *짧게* 끼어드는 형태입니다
(케이스 4와 반대 비율).

[출력 규칙]
- 형식: "사용자: ..." / "AI: ..." 줄바꿈으로 구분
- 마크다운 헤더, 굵게, 리스트 등 추가 서식 금지
- 코드 블록은 ``` 으로 감싸기 (단, 이 케이스에서는 코드가 거의 없거나 없음)
- 출력 전체를 하나의 코드 블록으로 감싸서 raw 텍스트로 보여주세요
- 다른 설명 없이 대화만
- **[기술], [잡담] 같은 라벨은 출력에 포함하지 마세요**

[턴 수 엄수 규칙]
- 사용자 발화는 정확히 10개. 9개나 11개는 모두 위반.
- 그중 잡담 발화는 정확히 7개여야 합니다.
- 위 흐름 가이드의 1턴~10턴이 사용자 발화 1번부터 10번까지에 1:1로 대응.
- 흐름 가이드의 어떤 턴도 다른 턴과 합치거나 생략하지 마세요.
- 대화를 작성하는 동안 속으로 사용자 발화 번호와 (기술/잡담)을 카운트하세요.
- 출력 직전에 다음을 확인하세요:
  - 사용자 발화 개수 = 10
