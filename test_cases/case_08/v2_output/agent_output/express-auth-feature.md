# Express 회원가입·로그인 기능

Express 서버에 회원가입과 로그인 기능을 만든다. 가입 시점에는 bcrypt로 비밀번호를 단방향 해시로 안전하게 저장하고, 로그인 이후에는 JWT로 stateless하게 사용자 상태를 증명한다. 두 기술은 시점도 책임도 분리되어 있다.

## 윤곽

- **회원가입**: 이메일/비밀번호를 받아 bcrypt 해시로 변환해 저장.
- **로그인**: 이메일로 사용자 조회 → `bcrypt.compare`로 비밀번호 검증 → 통과 시 `jwt.sign`으로 토큰 발급 → 클라이언트에 응답.
- **이후 요청**: 클라이언트가 Authorization 헤더에 토큰을 실어 보냄 → 서버는 `jwt.verify` 미들웨어로 검증하고 `req.user`에 사용자 정보를 꽂아 줌.

핵심 호출: 가입 = `bcrypt.hash`, 로그인 = `bcrypt.compare` + `jwt.sign`, 이후 요청 = `jwt.verify`.

## 참조 매핑

| 사용된 지식 | 참조 이유 |
|---|---|
| [bcrypt](./bcrypt.md) | 비밀번호를 평문으로 저장하면 DB 유출 시 모든 사용자가 위험해진다. 가입 시 단방향 해시로 저장하고 로그인 시 `compare`로 검증하기 위해 사용. saltRounds는 자기 서버에서 측정해 200~300ms 범위로 선택. |
| [JWT 인증](./jwt-auth.md) | 로그인 이후 매 요청마다 비밀번호를 다시 받을 수 없으므로 사용자 상태를 유지해야 한다. 서버 저장소 없이 서명만으로 검증되는 stateless 특성이 SPA·모바일 확장에 적합해 JWT를 채택. 토큰 만료와 저장 위치(httpOnly 쿠키 권장)로 보안 강도 조정. |
