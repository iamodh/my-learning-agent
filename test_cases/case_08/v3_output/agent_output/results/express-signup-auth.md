# Express 회원가입·로그인 인증 구조

Express 서버에 이메일/비밀번호 기반 회원가입 기능을 만들고, 로그인 성공 후 JWT로 사용자 상태를 유지하는 구조를 만든다.

## 만들려는 것

- 회원가입 시 이메일과 비밀번호를 받는다.
- 비밀번호는 평문이 아니라 bcrypt 해시로 저장한다.
- 로그인 시 저장된 해시와 입력 비밀번호를 bcrypt로 비교한다.
- 검증이 통과하면 JWT를 발급한다.
- 이후 요청은 Authorization 헤더의 토큰을 서버가 검증해 인증한다.

## 참조 지식

- `knowledge/bcrypt-password-hashing.md`: 가입 시 비밀번호를 안전하게 저장하고 로그인 시 검증하기 위해 필요하다.
- `knowledge/jwt-authentication.md`: 로그인 이후 매번 비밀번호를 입력하지 않고 사용자 상태를 유지하기 위해 필요하다.

## 핵심 흐름

1. 클라이언트가 이메일과 비밀번호를 서버에 보낸다.
2. 가입 시 서버는 `bcrypt.hash`로 비밀번호를 해싱해 DB에 저장한다.
3. 로그인 시 서버는 이메일로 사용자를 조회한다.
4. `bcrypt.compare`로 입력 비밀번호와 저장된 해시를 비교한다.
5. 일치하면 `jwt.sign`으로 토큰을 발급한다.
6. 클라이언트는 이후 요청마다 Authorization 헤더에 토큰을 보낸다.
7. 서버는 `jwt.verify`로 토큰을 검증하고 `req.user`에 사용자 정보를 넣는다.
