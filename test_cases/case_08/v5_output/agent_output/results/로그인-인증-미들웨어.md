# 로그인 + 인증 미들웨어

`/login`에서 비밀번호를 검증하고 JWT를 발급한다. 이후 요청들은 인증 미들웨어가 토큰을 검증해 `req.user`에 사용자 정보를 꽂아준다.

## 전체 흐름

클라이언트가 이메일/비밀번호 전송 → 서버가 이메일로 DB에서 사용자 조회 → `bcrypt.compare`로 입력 비밀번호와 저장된 해시 비교 → 일치하면 `jwt.sign`으로 토큰 발급 → 클라이언트는 이후 요청마다 `Authorization` 헤더에 토큰을 실어 보냄 → 서버는 `verify` 미들웨어로 검증하고 `req.user`에 사용자 정보를 꽂아준다.

## 구현

```js
const bcrypt = require('bcrypt');
const jwt = require('jsonwebtoken');
const SECRET = process.env.JWT_SECRET;

// 로그인
app.post('/login', async (req, res) => {
  const { email, password } = req.body;
  const user = await User.findOne({ where: { email } });
  const valid = await bcrypt.compare(password, user.password);
  if (!valid) return res.status(401).json({ error: 'invalid' });

  const token = jwt.sign(
    { userId: user.id, email: user.email },
    SECRET,
    { expiresIn: '1h' }
  );
  res.json({ token });
});

// 인증 미들웨어
function authenticate(req, res, next) {
  const token = req.headers.authorization?.split(' ')[1];
  try {
    const payload = jwt.verify(token, SECRET);
    req.user = payload;
    next();
  } catch (e) {
    res.status(401).json({ error: 'invalid token' });
  }
}
```

payload에는 식별자(userId, email)만 넣고 민감 정보는 제외한다.

| `knowledge/bcrypt-비밀번호-해싱.md` | 로그인 시 입력 비밀번호와 저장된 해시를 `bcrypt.compare`로 비교해 검증하기 위해 사용. |
