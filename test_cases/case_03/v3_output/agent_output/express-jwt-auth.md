# Express 서버 JWT 인증

만들려는 것은 기존에 라우터만 있는 Express 서버에 JWT 인증을 붙이는 것이다. 적용할 지식은 JWT 발급과 검증, 액세스 토큰과 리프레시 토큰을 함께 쓰는 인증 흐름이다.

## 만들 대상

Express 서버에서 로그인 성공 시 토큰을 발급하고, 보호된 라우터 앞에서 요청의 `Authorization` 헤더에 담긴 토큰을 검증하는 인증 구조를 만든다.

전체 흐름은 다음 네 단계로 정리됐다.

1. 서버가 토큰을 발급한다.
2. 클라이언트가 토큰을 저장한다.
3. 클라이언트가 매 요청마다 토큰을 같이 보낸다.
4. 서버가 토큰을 검증한다.

## 참조 지식

- `jwt.md`: 로그인 성공 시 액세스 토큰을 발급하고, 페이로드에는 사용자 식별에 필요한 최소 정보만 넣기 위해 필요하다.
- `refresh-token.md`: 액세스 토큰이 만료됐을 때 사용자를 매번 다시 로그인시키지 않고 새 액세스 토큰을 발급하기 위해 필요하다.

## 최종 구조

대화에서 정리된 최종 구조는 다음과 같다.

- 액세스 토큰은 짧게 유지해 탈취 피해 시간을 줄인다.
- 리프레시 토큰은 긴 수명으로 두되 DB에서 추적 가능하게 관리한다.
- 리프레시 토큰은 회전시켜 이전 토큰을 폐기한다.
- 페이로드는 노출될 수 있으므로 식별자 중심으로만 구성한다.
- 액세스 토큰은 서명 검증을 하고, 리프레시 토큰은 서명 검증과 DB 검증을 함께 한다.
- 액세스 토큰용 시크릿과 리프레시 토큰용 시크릿은 분리한다.
