# Express 서버 JWT 인증

라우터만 만들어둔 Express 서버에 JWT 기반 인증을 붙이는 것이 목표였다. 구조는 짧은 수명의 액세스 토큰으로 평소 API를 인증하고, 만료되면 긴 수명의 리프레시 토큰으로 재발급받는 형태다. 리프레시 토큰은 DB로 추적·회전시켜 탈취 위험을 분산시킨다.

## 구성 요소

- `jsonwebtoken` 라이브러리.
- 환경변수에 액세스/리프레시용 시크릿 분리.
- `/login`: 자격 확인 후 액세스 토큰 발급(`expiresIn: '15m'`).
- `/refresh`: 쿠키에 담긴 리프레시 토큰을 `jwt.verify`로 검증 + DB 조회로 살아있는지 확인 후 새 액세스 토큰 발급.
- 보호된 라우터 앞에 미들웨어를 두고 Authorization 헤더의 토큰을 검증.
- 리프레시 토큰은 httpOnly 쿠키로 클라이언트에 저장, 서버 DB에서 발급·폐기 관리, 회전(rotation)으로 탈취 대응.

## 참조 지식

| 지식 | 참조 이유 |
|---|---|
| [JWT](./jwt.md) | 인증 토큰의 발급·검증 메커니즘 전체(구조, 서명 vs 암호화, 페이로드 원칙, 액세스/리프레시 토큰 전략, 회전 방어)가 이 결과물의 기반이기 때문. |
