# 배포: Vercel vs Netlify

## 핵심 결론
Next.js는 Vercel(Next.js 제작사)이 가장 잘 지원한다. 호환성 신경 안 쓰고 모든 기능을 쓸 수 있어 학습 시 변수가 줄어든다 → Vercel 선택.

## Vercel
- Next.js 기능이 별도 설정 없이 대부분 그대로 동작.
- GitHub 연동 시 푸시할 때마다 자동 배포.
- 개인 블로그 규모는 무료 플랜으로 충분.

## Netlify (비교 검토)
- 정적 사이트 쪽 오래된 강자, 기능 자체는 비슷.
- 무료 플랜 빌드 시간 한도가 더 넉넉하고 폼/함수 등 자체 기능이 잘 다듬어져 있음.
- Next.js 런타임 어댑터로 기본 SSG/ISR/라우팅은 동작하나, 신규 기능은 Vercel에 먼저 들어왔다가 따라오는 구조라 한 박자 늦거나 동작이 미묘하게 다를 수 있음.

## 의사결정 기록
처음엔 가격 때문에 Netlify에 끌렸으나, Next.js 1급 지원과 학습 단순화를 우선해 최종적으로 Vercel로 결정.
