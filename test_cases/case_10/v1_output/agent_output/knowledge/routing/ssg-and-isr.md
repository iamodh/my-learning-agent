# SSG와 ISR (정적 생성)

## 핵심
- 블로그 글은 빌드 시점에 미리 정적 HTML로 만들어두는 SSG가 잘 맞는다.
- `generateStaticParams`를 export해 모든 `slug` 목록을 반환하면, 빌드 시 각 slug마다 페이지가 정적 생성된다.
- 기본적으로 새 글이 추가되면 다시 빌드/배포해야 반영된다.

## ISR
- `export const revalidate = 60` 처럼 적으면 일정 시간(초)마다 백그라운드에서 페이지를 재생성한다.
- 개인 블로그는 글 올릴 때마다 푸시해 재빌드하는 방식이 보통 더 깔끔하다.

## 정리
- SSG: 빌드 시점 생성 → 빠르고 단순, 갱신은 재빌드 필요.
- ISR: 주기적 백그라운드 재생성 → 재배포 없이 갱신 가능하나 복잡도 증가.
