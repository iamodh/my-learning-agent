# Vercel 배포와 환경 변수

Vercel은 Next.js를 만든 회사라서, Next.js 기능들이 별도 설정 없이 거의 다 그냥 동작하는 게 강점이다. GitHub 연동해두면 푸시할 때마다 자동 배포된다. 개인 블로그 규모면 무료 플랜으로 충분하다.

## 환경 변수 분리

로컬과 배포 환경에서 환경 변수를 분리해 쓴다.

- 로컬: 프로젝트 루트에 `.env.local` 파일을 만들고 `API_KEY=...` 같은 형태로 적는다.
- 배포: Vercel 대시보드의 Settings → Environment Variables에서 같은 키를 등록한다.

브라우저에 노출돼야 하는 값은 키 이름에 `NEXT_PUBLIC_` 접두사를 붙여야 한다.

## 검토했으나 채택하지 않은 대안 — Netlify

Netlify도 정적 사이트 쪽에선 오래된 강자이고 무료 플랜 빌드 시간 한도가 더 넉넉한 편이다. Netlify에 Next.js 런타임 어댑터가 있어 기본적인 SSG·ISR·라우팅은 잘 돈다. 다만 Next.js 신규 기능이 Vercel에 먼저 들어왔다가 어댑터로 따라오는 구조라 한 박자 늦거나 일부 동작이 미묘하게 다를 수 있다. 호환성 변수를 줄이기 위해 Vercel을 선택했다.
