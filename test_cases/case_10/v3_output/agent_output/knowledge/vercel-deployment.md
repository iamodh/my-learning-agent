# Vercel 배포와 환경 변수

Vercel은 Next.js를 만든 회사의 배포 플랫폼이라 Next.js 기능들이 별도 설정 없이 거의 그대로 동작하는 선택지로 정리됐다. GitHub와 연동하면 푸시할 때마다 자동 배포할 수 있고, 개인 블로그 규모에서는 무료 플랜으로도 충분하다고 언급됐다.

## 환경 변수

로컬에서는 프로젝트 루트의 `.env.local` 파일에 환경 변수를 적는다.

```env
API_KEY=...
```

Vercel에서는 대시보드 Settings의 Environment Variables에 같은 키를 등록한다.

브라우저에 노출되어야 하는 값은 `NEXT_PUBLIC_` 접두사를 붙여야 한다.

## Netlify 검토 결과

Netlify도 정적 사이트 배포에서는 기능이 비슷하고, 무료 플랜 한도나 폼, 함수 기능이 언급됐다. 다만 Next.js 특유 기능은 Vercel만큼 1급 지원은 아니며, 새로운 기능이 어댑터를 통해 뒤따라오는 구조라고 정리됐다.

최종 배포 선택은 Vercel이다.
