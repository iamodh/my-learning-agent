# Vercel 배포와 환경 변수

Vercel은 Next.js를 만든 회사가 운영하는 배포 플랫폼이라, 별도 설정 없이 Next.js의 기능 대부분이 그대로 동작한다. GitHub 연동을 해두면 푸시할 때마다 자동으로 배포되며, 개인 블로그 규모면 무료 플랜으로 충분하다.

## 환경 변수

로컬과 배포 환경의 환경 변수는 다음과 같이 관리한다.

- 로컬: 프로젝트 루트에 `.env.local` 파일을 만들어 `API_KEY=...` 형태로 적는다.
- Vercel: 대시보드 Settings의 Environment Variables에 같은 키를 등록한다.
- 브라우저에 노출돼야 하는 값은 키 이름 앞에 `NEXT_PUBLIC_` 접두사를 붙인다.

## Netlify 비교 (탈락한 대안)

초기에는 Netlify도 검토했다. 무료 플랜의 빌드 시간 한도가 좀 더 넉넉하고, 폼·함수 같은 자체 기능이 잘 다듬어져 있다는 장점이 있다. 다만 Next.js의 새로운 기능들은 Vercel 쪽에 먼저 들어왔다가 어댑터로 따라오는 구조라, Netlify에서는 한 박자 늦거나 일부 동작이 미묘하게 달라질 수 있다. 단순 정적 블로그면 차이가 거의 안 느껴질 수도 있지만, 학습 중 호환성 변수를 줄이려고 최종적으로 Vercel을 선택했다.
