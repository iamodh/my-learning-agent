# Vercel 배포

Vercel은 Next.js를 만든 회사가 운영하는 호스팅 플랫폼으로, Next.js 기능들이 별도 설정 없이 거의 그대로 동작한다. GitHub 연동을 해두면 푸시할 때마다 자동 배포된다. 개인 블로그 규모면 무료 플랜으로 충분하다.

## 환경 변수

로컬과 배포 환경의 값을 분리해 관리한다.

- 로컬: 프로젝트 루트에 `.env.local`을 만들어 `API_KEY=...` 같이 적어둔다.
- Vercel: 대시보드 Settings의 Environment Variables에서 같은 키를 등록한다.
- 브라우저에 노출돼야 하는 값은 `NEXT_PUBLIC_` 접두사를 붙인다.

## 탈락한 대안: Netlify

Netlify도 정적 사이트 호스팅의 오랜 강자고, 무료 플랜의 빌드 시간 한도가 더 넉넉하며 폼·함수 같은 자체 기능이 잘 다듬어져 있다. Next.js 런타임 어댑터가 있어 기본적인 SSG·ISR·라우팅도 돌아간다.

다만 Next.js의 새 기능은 Vercel에 먼저 들어왔다가 어댑터로 따라오는 구조라, 한 박자 늦거나 동작이 미묘하게 달라질 수 있다. **이번 블로그에서는 Next.js 기능을 변수 없이 다 쓰기 위해 Vercel을 선택했고, Netlify는 채택하지 않았다.**
