# 외부 API 응답 캐싱

서비스에서 거의 바뀌지 않는 외부 API 응답을 매번 호출해 비용과 지연이 늘어나는 문제가 있었다. 이를 줄이기 위해 Redis를 사용해 외부 API 응답을 캐싱하는 방향으로 정리했다.

## 만들려는 것

Express 기반 API에서 외부 API 호출 결과를 Redis에 저장하고, 같은 요청이 다시 들어오면 Redis 캐시를 먼저 조회해 응답하는 캐싱 구조를 만든다.

## 참조 지식

- `redis-caching.md`

## 참조 이유

Redis 기반 캐싱의 기본 조회 흐름, TTL 설정, 키 네이밍, 데이터 변경 시 `del`을 통한 무효화, 캐시 스탬피드 대응이 모두 이 구현에 직접 필요하다.

## 적용 방향

요청이 들어오면 `product:{id}` 같은 계층형 키로 Redis를 먼저 조회한다. 캐시가 있으면 JSON으로 파싱해 반환하고, 없으면 외부 API를 호출한 뒤 `EX 300`처럼 TTL을 붙여 Redis에 저장한다.

데이터가 변경되는 시점에는 TTL 만료를 기다리지 않고 해당 캐시 키를 삭제한다. 인기 키 만료 시 외부 API 호출이 몰릴 수 있으므로, 트래픽이 큰 키에는 짧은 락을 두어 한 요청만 새 값을 채우게 하는 방식을 고려한다.
