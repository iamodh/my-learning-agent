# Redis 동작 원리

Redis는 데이터를 디스크가 아니라 메모리(RAM)에 키-값 형태로 저장하는 데이터 스토어다. 디스크 조회나 외부 API 호출보다 훨씬 빠르며, 응답이 보통 마이크로초 단위라 캐시 용도로 적합하다.

## 특성

- **인메모리 키-값 저장**: 모든 데이터를 RAM에 키-값 쌍으로 보관한다.
- **단일 스레드 명령 처리**: 명령을 순차적으로 처리해서 동시성 문제가 단순한 편이다.
- **별도 프로세스**: 애플리케이션 메모리 변수와 달리 별도 프로세스로 떠 있어서, 여러 서버 인스턴스가 같은 캐시를 공유할 수 있다.

## 캐시로 쓰는 기본 흐름

요청이 들어오면 Redis에서 키로 먼저 조회하고, 있으면 그대로 반환(캐시 히트), 없으면 원본(외부 API·DB 등)을 호출해서 결과를 Redis에 저장한 뒤 반환한다(캐시 미스).
