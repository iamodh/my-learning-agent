# 키 네이밍 컨벤션

캐시 키는 콜론(`:`)으로 계층을 구분하는 컨벤션을 쓴다. `리소스:식별자:변형(파라미터)` 순서로 적는다.

## 예시

- `product:123`
- `user:42:profile`
- `search:keyword:laptop:page:2`

## 이렇게 쓰는 이유

- 나중에 `KEYS product:*`나 `SCAN`으로 패턴 매칭해서 일괄 처리할 수 있다.
- 다른 개발자가 봐도 키가 무엇을 가리키는지 바로 알 수 있다.
- 환경별 prefix를 붙이는 것도 흔하다 — `prod:product:123` 처럼.

`data1`, `userdata` 같은 평탄한 이름은 충돌·관리 문제가 생기기 쉽다.
