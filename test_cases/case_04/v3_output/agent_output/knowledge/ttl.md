# TTL과 만료 처리

TTL(Time To Live)은 캐시 키가 유지되는 시간이다. 지정한 시간이 지나면 Redis가 키를 자동으로 만료시킨다.

## TTL 값을 정하는 기준

데이터가 얼마나 자주 바뀌는지에 맞춘다.

- 거의 안 바뀌는 메타데이터: 1시간 정도
- 자주 바뀌는 데이터: 30초~5분 정도

## 내부 만료 메커니즘

Redis는 키마다 만료 시간을 기록해두고 두 가지 방식을 함께 쓴다.

- **Lazy expiration**: 클라이언트가 그 키에 접근할 때 만료됐는지 체크하고 삭제한다.
- **Active expiration**: 백그라운드에서 주기적으로 키를 랜덤 샘플링해 만료된 것을 청소한다.

그래서 TTL이 지난 키가 잠깐 메모리에 남아 있을 수는 있지만, 조회하면 무조건 없는 것으로 처리된다.
