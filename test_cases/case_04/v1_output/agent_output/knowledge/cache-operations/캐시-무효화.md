# 캐시 무효화 (Cache Invalidation)

## 핵심
데이터가 바뀌면 TTL 만료만 기다리지 말고, 변경 시점에 **명시적으로 캐시를 무효화**해야 최신 데이터를 보장할 수 있다.

## 두 가지 전략
1. **삭제 방식 (권장)**: DB 업데이트 직후 `redis.del(key)`로 키를 지운다. 다음 조회 때 자연스럽게 캐시 미스가 나서 새 데이터로 다시 채워진다.
2. **덮어쓰기 방식**: 업데이트하면서 캐시도 새 값으로 덮어쓴다.

## 왜 삭제가 더 안전한가
덮어쓰기 도중에 다른 요청이 동시에 옛날 값을 또 써버리는 **race condition**이 생길 수 있다.
삭제는 "다음에 누가 읽을 때 최신으로 다시 받아와"라는 단순한 의도라 동시성으로 꼬일 일이 적다.

## 예시
```javascript
// DB 업데이트 후 해당 키 무효화
await db.updateProduct(id, newData);
await redis.del(`product:${id}`);
```
