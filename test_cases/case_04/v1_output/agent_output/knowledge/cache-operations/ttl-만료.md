# TTL과 캐시 만료

## 핵심
TTL(Time To Live)은 캐시 키의 생존 시간이다. 캐시가 영원히 살아있으면 오래된 데이터를 계속 반환하므로, **데이터가 바뀌는 주기에 맞춰** TTL을 정한다.

## TTL을 정하는 기준
- 거의 안 바뀌는 메타데이터: 1시간 정도
- 자주 바뀌는 데이터: 30초 ~ 5분 정도

## 만료 시 Redis 내부 동작
Redis는 키마다 만료 시간을 기록해두고, 두 가지 방식으로 만료된 키를 정리한다.

- **Lazy expiration**: 클라이언트가 그 키에 접근할 때 만료 여부를 체크해서 삭제한다.
- **Active expiration**: 백그라운드에서 주기적으로 랜덤 샘플링하여 만료된 키를 청소한다.

TTL이 지난 키가 잠깐 메모리에 남아있을 수는 있지만, **조회하면 무조건 없는 것으로 처리**되므로 정확성에는 문제가 없다.

## 예시
```javascript
// EX 300 = TTL 300초 (5분 뒤 자동 삭제)
await redis.set(key, JSON.stringify(data), 'EX', 300);
```
