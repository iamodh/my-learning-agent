# 캐시 키 네이밍 컨벤션

## 핵심
콜론(`:`)으로 계층을 구분해 **`리소스종류:식별자:변형`** 순서로 키를 만든다. `data1`, `userdata`처럼 즉흥적으로 지으면 충돌과 관리 문제가 생긴다.

## 패턴 예시
- `product:123`
- `user:42:profile`
- `search:keyword:laptop:page:2`

리소스 종류 → 식별자 → 변형(파라미터) 순으로 넣는다.

## 이렇게 하는 이유
- `KEYS product:*`나 `SCAN`으로 **패턴 매칭하여 일괄 처리**할 수 있다.
- 다른 개발자가 봐도 키가 무엇을 가리키는지 바로 알 수 있다(가독성).

## 환경별 prefix
환경을 구분하기 위해 prefix를 붙이는 것도 흔하다.
- 예: `prod:product:123`
