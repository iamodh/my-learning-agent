# Redis 동작 원리

## 핵심
Redis는 데이터를 **디스크가 아닌 메모리(RAM)**에 키-값 형태로 보관한다. 그래서 디스크 조회나 외부 API 호출보다 훨씬 빠르고(보통 응답이 마이크로초 단위), 캐시 용도로 적합하다.

## 일반 메모리 변수와의 차이
- **별도 프로세스**로 동작한다. 따라서 여러 서버 인스턴스가 같은 캐시를 공유할 수 있다.
- 애플리케이션 메모리 변수는 인스턴스마다 따로 존재하지만, Redis는 공유 캐시가 된다.

## 동시성
명령을 **단일 스레드**로 순차 처리한다. 덕분에 동시성 문제가 비교적 단순하다.

## 캐시 기본 흐름 (Cache-Aside)
1. 요청이 들어오면 먼저 Redis에서 키로 조회한다.
2. 값이 있으면 그대로 반환한다 → **캐시 히트**
3. 값이 없으면 외부 API를 호출하고, 결과를 Redis에 저장한 뒤 반환한다 → **캐시 미스**
