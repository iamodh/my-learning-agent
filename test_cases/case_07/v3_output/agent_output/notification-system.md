# 실시간 알림 시스템

서버에서 클라이언트로 실시간 알림을 보내고, 클라이언트가 알림에 대한 `읽음` 액션도 서버로 보낼 수 있는 알림 시스템을 만들려 했다.

최종 방향은 WebSocket 기반의 양방향 통신이며, 라이브러리는 `socket.io`를 쓰는 쪽으로 정했다. 초기에는 SSE도 검토했으나, 클라이언트에서 서버로 `읽음` 표시를 보내야 해서 최종 선택에서는 제외했다.

## 참조 지식

- `knowledge/websocket.md`: 알림 수신과 읽음 처리처럼 서버와 클라이언트가 모두 메시지를 보내야 하므로 WebSocket의 양방향 연결이 필요하다.
- `knowledge/websocket-library-choice.md`: 알림 시스템에서 룸, 자동 재연결, 폴백 기능이 도움이 되므로 `socket.io`를 선택했다.
