# SSE

SSE(Server-Sent Events)는 서버가 클라이언트로 이벤트를 스트림처럼 보내는 방식이다. 별도 프로토콜이 아니라 HTTP 응답을 닫지 않고 유지한 채, 서버가 계속 이벤트를 흘려보낸다.

## 특징

SSE는 서버에서 클라이언트로만 데이터를 보내는 단방향 방식이다. 브라우저에서는 `EventSource` API로 받을 수 있고, HTTP 위에서 동작하므로 구현이 단순하고 HTTP 환경과 잘 맞는다.

서버는 응답의 `Content-Type`을 `text/event-stream`으로 설정하고 응답을 닫지 않는다. 이벤트는 `data: ...\n\n` 형식으로 전달된다.

연결이 끊기면 클라이언트가 자동 재연결을 시도할 수 있고, `Last-Event-ID`로 어디까지 받았는지 서버에 알려줄 수도 있다.

## 이번 대화에서 탈락한 이유

SSE 자체는 클라이언트에서 서버로 메시지를 보낼 수 없다. 알림을 받기만 하는 경우에는 적합하지만, 이번 시나리오에서는 클라이언트가 `읽음` 표시를 서버로 보내야 했다.

따라서 SSE를 쓰려면 수신은 SSE, 전송은 별도 HTTP POST로 나누어야 한다. 최종적으로는 이 분리 대신 WebSocket 기반의 양방향 통신을 선택했다.
