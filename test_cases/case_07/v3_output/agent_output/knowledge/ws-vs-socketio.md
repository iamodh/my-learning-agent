# ws vs socket.io

Node 환경에서 WebSocket을 다루는 대표적인 두 라이브러리의 차이를 정리한다.

## ws

WebSocket 프로토콜만 얇게 구현한 라이브러리. 가볍고 표준에 충실하다.

## socket.io

ws 위에 다음 기능을 얹어 놓은 프레임워크에 가까운 라이브러리.

- **룸(room)**: 연결을 그룹으로 묶어 브로드캐스트할 수 있다.
- **네임스페이스**: 하나의 연결 위에 논리적 채널을 나눌 수 있다.
- **이벤트 기반 API**.
- **자동 재연결**.
- **HTTP long-polling 폴백**: WebSocket이 안 될 때 자동으로 대체 전송을 사용한다.

단, socket.io는 자체 프로토콜을 쓰기 때문에 클라이언트도 `socket.io-client`를 써야 한다.

## 선택 기준

- 룸 단위 브로드캐스트, 재연결 처리가 자주 필요하면 socket.io가 편하다.
- 가볍게 표준 그대로 쓰고 싶으면 ws가 좋다.
