# WebSocket

WebSocket은 클라이언트와 서버가 하나의 TCP 연결을 지속적으로 유지하면서 양방향으로 메시지를 주고받을 수 있는 프로토콜이다. HTTP의 요청-응답 구조와 달리 서버가 먼저 클라이언트로 데이터를 보낼 수 있어 실시간 통신이 가능하다.

## HTTP와의 차이

HTTP는 요청-응답이 끝나면 연결이 끊기는 구조여서, 서버가 먼저 클라이언트에게 말을 걸 수 없다. WebSocket은 처음에 HTTP로 핸드셰이크(Upgrade 헤더)를 한 뒤, 그 TCP 연결을 그대로 유지하면서 별도 프로토콜(`ws://`)로 전환한다. 연결이 계속 살아 있기 때문에 서버가 원할 때 즉시 클라이언트로 데이터를 보낼 수 있고, 이것이 "실시간"이 가능한 이유다.

## 동작 흐름

1. 클라이언트가 HTTP로 핸드셰이크 요청(Upgrade 헤더 포함)을 보낸다.
2. 서버가 수락하면 같은 TCP 연결이 WebSocket 프로토콜로 전환된다.
3. 이후 양쪽이 자유롭게 메시지를 송수신한다.
