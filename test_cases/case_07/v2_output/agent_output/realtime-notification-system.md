# 실시간 알림 시스템

서버에서 클라이언트로 알림을 실시간으로 푸시하고, 클라이언트가 '읽음' 표시를 서버로 다시 보낼 수 있는 양방향 알림 시스템을 Express 환경에서 구축하고자 한다.

## 결정 사항

- **통신 방식**: WebSocket
- **라이브러리**: socket.io
- **서버 프레임워크**: Express

초기에는 SSE를 가벼운 대안으로 검토했으나, '읽음' 등 클라이언트 → 서버 액션이 필요해 최종적으로 WebSocket으로 결정했다.

## 참조 지식 및 참조 이유

| 지식 | 참조 이유 |
|------|-----------|
| [WebSocket](./knowledge/websocket.md) | 알림 수신(서버 → 클라이언트)과 '읽음' 처리(클라이언트 → 서버)를 한 연결로 처리하는 양방향 채널이 필요했다. 추후 채팅·입력 상태 등으로 확장될 가능성이 있어 처음부터 양방향 통신으로 통일하는 편이 메시지 순서·세션 관리 측면에서 일관적이다. |
| [WebSocket § ws vs socket.io](./knowledge/websocket.md#ws-vs-socketio) | 알림 시스템에서는 룸 단위 브로드캐스트, 자동 재연결, WebSocket 불가 시 long-polling 폴백 같은 운영 편의가 자주 필요해 socket.io의 부가 기능이 시나리오와 잘 맞는다. |
