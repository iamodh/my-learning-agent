# 실시간 알림 시스템 (Express + socket.io)

서버에서 클라이언트로 알림을 실시간 푸시하면서, 클라이언트의 "읽음" 처리 같은 액션도 같은 채널로 받기 위한 알림 시스템. Express 서버에 socket.io를 붙여 양방향 채널 하나로 통일한다.

## 무엇을 만들었는가

- Express 기반 서버에 WebSocket 채널을 얹어 서버 → 클라이언트 알림 푸시와 클라이언트 → 서버 "읽음" 이벤트를 하나의 연결로 처리.
- 추후 채팅이나 입력 상태 같은 양방향 기능으로 확장될 가능성도 같은 채널 위에서 흡수.

## 사용된 지식과 참조 이유

- **[WebSocket](./websocket.md)** — 단일 TCP 연결로 서버 → 클라이언트 푸시와 클라이언트 → 서버 액션("읽음")을 모두 처리해야 하므로 양방향 프로토콜이 필요했다. 라이브러리는 룸 단위 브로드캐스트, 자동 재연결, WebSocket 미지원 환경에서의 long-polling 폴백 때문에 `ws` 대신 **socket.io**를 선택.

> 초기에는 더 가벼운 SSE를 검토했으나, "읽음" 같은 클라이언트 → 서버 액션이 필요해 양방향을 지원하는 WebSocket으로 결정.

## 구현 방향

- Express 앱과 동일한 http 서버에 socket.io를 붙여 단일 포트에서 운영.
- 서버 → 클라이언트: 알림 발생 시 해당 사용자의 room으로 알림 이벤트 발신.
- 클라이언트 → 서버: "읽음" 이벤트를 같은 socket 연결로 송신.
- socket.io의 자동 재연결과 long-polling 폴백으로 연결 안정성 확보.
