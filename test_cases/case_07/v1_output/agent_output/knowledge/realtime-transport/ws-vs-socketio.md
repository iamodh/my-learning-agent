# ws vs socket.io 라이브러리 선택

## 핵심
- **ws**: WebSocket 프로토콜만 얇게 구현. 가볍고 표준에 충실.
- **socket.io**: ws 위에 룸/네임스페이스/자동 재연결/이벤트 API/폴백을 얹은 프레임워크.

## 비교

| 항목 | ws | socket.io |
|------|-----|-----------|
| 성격 | 표준 WebSocket 얇은 구현 | 기능 확장 프레임워크 |
| 룸/네임스페이스 | 없음 (직접 구현) | 내장 |
| 자동 재연결 | 없음 | 내장 |
| 폴백 | 없음 | WebSocket 불가 시 HTTP long-polling 자동 폴백 |
| 클라이언트 | 표준 WebSocket 클라이언트 | `socket.io-client` 필요 (자체 프로토콜) |

## 판단 가이드
- 룸 단위 브로드캐스트, 재연결 처리가 자주 필요 → socket.io가 편함.
- 가볍게 표준 그대로 가고 싶음 → ws.
