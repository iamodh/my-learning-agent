# 사이드 프로젝트 OAuth 로그인 기능

## 만들려는 것

사이드 프로젝트에 이메일/비밀번호를 직접 관리하지 않는 로그인 기능을 붙인다. 방향은 구글이나 깃허브 OAuth를 사용하는 것이며, 대화에서는 구글 OAuth 예시를 중심으로 흐름을 잡았다.

## 참조 지식

- `knowledge/oauth-login.md`
- `knowledge/session-management.md`

## 참조 이유

OAuth 지식은 사용자를 외부 인증 제공자에게 보내고, 콜백에서 인가 코드를 access token으로 교환한 뒤 사용자 프로필을 가져오는 로그인 흐름을 구현하기 위해 필요하다.

세션 관리 지식은 OAuth로 사용자 정보를 받은 뒤 매 요청마다 다시 구글에 묻지 않고, 서버가 세션 ID와 쿠키를 통해 로그인 상태를 유지하기 위해 필요하다.
