# 세션 관리

세션 관리는 OAuth로 사용자를 확인한 뒤, 이후 요청에서 서버가 같은 사용자를 계속 식별하기 위한 상태 유지 방식이다.

대화에서는 OAuth 콜백에서 사용자 정보를 받은 뒤 서버가 세션 ID를 발급하고, 이를 HttpOnly 쿠키로 브라우저에 심는 방식이 설명됐다. 서버는 Redis나 DB 같은 세션 저장소에 `{ sessionId -> userId }` 관계를 보관하고, 이후 요청마다 쿠키의 세션 ID로 사용자를 식별한다.

```js
req.session.userId = profile.email; // express-session 기준
```

## 세션 방식 선택

JWT 방식도 언급됐지만, 이 케이스에서는 만료와 강제 로그아웃 제어가 더 직관적인 세션 방식이 추천됐다.

## 만료 전략

세션 만료는 두 전략을 섞어 사용할 수 있다고 설명됐다.

- sliding expiration: 요청이 올 때마다 만료 시각을 갱신해 활동 중인 사용자가 끊기지 않게 하는 방식
- absolute expiration: 활동 여부와 관계없이 최초 로그인 후 정해진 시간으로 상한을 두는 방식

대화에서는 일반 서비스 기준 예시로 sliding 30분과 absolute 14일이 언급됐다. 결제 같은 민감 액션 직전에는 더 짧은 재인증을 따로 요구할 수 있다고 했다.

Redis 같은 저장소를 쓰면 키에 TTL을 걸어 만료를 처리할 수 있다.
