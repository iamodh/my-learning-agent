# 문서 배치

사이드 프로젝트에 로그인과 결제 기능을 붙이기 위한 학습 산출물이다.

```
agent_output/
├── README.md
├── 사이드프로젝트-로그인-결제.md      # 결과 문서: 만들려는 것 + 참조 매핑
└── knowledge/
    ├── oauth-로그인.md                # 구글 OAuth 흐름 + Express 구현
    ├── 세션-관리.md                   # 세션 동작·만료 정책 (JWT는 탈락)
    └── 결제-처리.md                   # PG사/PCI DSS, 토스페이먼츠 confirm, 인증 미들웨어
```

- 결과 문서는 루트에 두고, 그 근거가 된 개념들은 `knowledge/`에 모았다.
- 세션 만료 정책은 세션 관리의 하위 측면이라 별도 파일로 분리하지 않고 섹션으로 통합했다.
- 결제 라우트 인증 미들웨어는 결제 흐름의 일부라 결제 처리 문서의 섹션으로 통합했다.
