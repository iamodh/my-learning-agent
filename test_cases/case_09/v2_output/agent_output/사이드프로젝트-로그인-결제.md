# 사이드 프로젝트 로그인 + 결제 기능

사이드 프로젝트에 로그인과 결제 기능을 함께 붙인다. 로그인은 구글 OAuth로 사용자를 식별하고, 결제는 토스페이먼츠로 처리한다. 두 흐름의 연결고리는 세션이다.

## 구성 개요

- **로그인**: 구글 OAuth로 사용자 확인 → 세션 발급 → HttpOnly 쿠키로 브라우저에 보관
- **결제**: 인증 미들웨어로 세션 검증 → 주문 소유자·금액 대조 → 토스페이먼츠 confirm 호출
- 세션 만료 정책(sliding 30분 + absolute 14일)이 양쪽 UX·보안에 모두 영향을 준다.

## 참조한 지식과 참조 이유

| 지식 문서 | 참조 이유 |
|---|---|
| `knowledge/oauth-로그인.md` | 비밀번호를 직접 관리하지 않고 구글로 사용자 식별을 위임하기 위해 채택. 콜백 → 토큰 교환 → 프로필 조회 흐름을 그대로 구현한다. |
| `knowledge/세션-관리.md` | OAuth로 받은 사용자 정보를 매 요청마다 다시 묻지 않기 위해 세션으로 상태를 유지. 만료·강제 로그아웃 컨트롤이 직관적이라 세션을 선택. 만료 정책으로 sliding + absolute 혼합 사용. |
| `knowledge/결제-처리.md` | 카드 정보 보안 규제(PCI DSS) 부담을 PG사로 넘기기 위해 토스페이먼츠 채택. 클라이언트 결제창 → 서버 confirm 패턴으로 위변조를 막고, 결제 라우트 앞에 인증 미들웨어를 두어 세션과 결합한다. |
