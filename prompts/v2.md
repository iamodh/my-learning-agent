# 학습 에이전트 프롬프트 v2

첨부한 대화를 기반으로 내가 어떤 지식을 학습하려고 했고, 이 지식을 통해 무엇을 만드려고 했는지 지식 문서와 결과 문서로 분류해줘.

## 1. 산출물 범위 (대화에서 실제로 확립된 것만)

- 산출물은 대화에서 명시적으로 다뤄진 내용으로 **엄격히 한정한다**. 대화에 없던 설정값·옵션·대안 비교·모범 사례는 추가하지 마라.
- 얕게 언급된 주제는 얕게 남긴다. 추측으로 상세화하거나 일반 지식을 끌어와 보강하지 마라.
- 잡담·여담은 어떤 문서의 본문에도 유입시키지 마라.

## 2. 지식 문서

지식 문서는 하나의 개념 범위여야 해. 핵심 내용은 앞에 배치하고, 첫 문단은 코드가 아니라 정의 텍스트로 시작해줘. 이해가 어려운 부분(특히 사용자가 명시적으로 예시·코드를 요청한 개념)에는 예시나 코드 스니펫을 넣어도 돼.

### 2.1 분할 vs 통합 기준 (중요)

문서를 쪼개기 전에 다음 기준을 적용한다. **애매하면 합친다.**

- 어떤 세부 주제가 부모 개념을 떼면 단독으로 의미가 성립하지 않거나, 사용자가 그 부모 개념을 학습하다 마주친 하위 측면이라면 → **새 파일이 아니라 부모 문서의 `##` 섹션으로 통합한다.**
- 새 지식 문서는 그 개념이 **서로 무관한 여러 결과·맥락에서 독립적으로 재사용될 때만** 정당하다. 한 결과 문서에서만 쓰이는 부수 개념은 섹션으로 둔다.
- 두 번째 이상의 관련 지식 파일을 만들기 전에, **그것이 기존 문서의 섹션이 될 수 없는지 먼저 자문하라.** 섹션으로 표현 가능하면 섹션으로 둔다.
- 기본 편향은 **문서 수 최소화**다. 쪼갤지 합칠지 판단이 서지 않으면 합쳐라.

### 2.2 탈락한 대안의 처리

사용자가 검토했다가 채택하지 않은 대안은 지식 문서로 남길 수 있으나, 그것이 *탈락했다는 사실*을 본문에 명시한다. 결과 문서의 참조 매핑에는 넣지 않는다(§3.2 참고).

## 3. 결과 문서 (조건부 산출물)

### 3.1 생성 조건

결과 문서는 **항상 만드는 산출물이 아니다.** 대화에 다음이 모두 성립할 때만 만든다.

- 사용자가 만들려는 대상에 대한 **명확한 제작 의도**가 있고,
- 그 대상의 윤곽(무엇을 만드는가)이 대화에서 식별 가능할 때.

대화가 주로 지식 습득이고 만들려는 대상이 가정·여담·스치는 언급 수준이면 **지식 문서만 만들고 결과 문서·참조 매핑·참조 이유를 생성하지 마라.**

### 3.2 결과 문서를 만들 때

- 결과 문서는 내가 만드려고 했던 것이 무엇인지 적고, 어떤 지식이 사용되었는지 참조해줘. 그리고 왜 그 지식이 사용되었는가 이유(참조 이유)를 포함해줘.
- 참조는 **최종 결정의 근거가 된 지식만** 포함한다. 검토 끝에 탈락한 대안은 참조 매핑에 넣지 마라.
- 사용자가 선택을 번복한 경우, 결정 과정을 시간순으로 재현하지 마라. 한 줄짜리 요약("초기에는 X를 검토했으나 최종적으로 Y로 결정") 정도로만 남긴다.

## 4. 길이와 형식

모든 문서는 간결한 길이를 유지해줘. 핵심을 앞에 두고, 곁가지 설명은 뒤로 미루거나 잘라낸다.

## 5. 폴더 구조

문서를 모두 생성한 뒤 인과 관계에 따라 폴더 구조를 만든다.

- 폴더 깊이는 3 이하로 제한한다. 다만 이는 상한일 뿐, **인과 관계를 표현하는 가장 얕은 구조**를 택한다.
- **파일이 1~2개뿐인 분류 폴더는 만들지 마라.** 그 정도는 상위 폴더에 평탄하게 둔다.
- 카테고리를 만들기 위해 카테고리를 만들지 마라. 묶을 실체가 있을 때만 폴더로 분리한다.

마지막으로 각 문서를 알맞은 폴더에 배치하고, 루트 디렉토리에 문서 배치 구조를 표시한 README를 만들어줘.

## 6. 자가 점검 (출력 직전)

문서를 확정하기 전에 스스로 확인한다.

1. 지식 문서 중 부모 개념의 섹션으로 통합 가능한 것이 있는가? 있으면 통합한다.
2. 결과 문서를 만들었다면, 대화에 명확한 제작 의도가 있었는가? 아니면 결과 문서·참조 매핑을 제거한다.
3. 참조 매핑에 탈락한 대안이 들어가 있지 않은가?
4. 대화에 없던 설정값·옵션·대안 비교가 본문에 섞이지 않았는가?
5. 파일 1~2개인 분류 폴더가 있지 않은가?
